# 公众号归档器

一个本地优先的 Skill：把微信公众号文章同步、浏览收藏和链接导入统一保存到本地资料库。

```text
汇总并列出所有已订阅公众号最近 50 篇文章，按公众号和日期整理
重新下载「<某公众号>」最新 20 篇文章，并整理到本地
下载「<某公众号>」最新 5 篇文章的评论和互动数据
开启微信收藏，我要边看公众号文章，边收藏有价值的文章和评论
把这些公众号文章链接导入本地：https://mp.weixin.qq.com/s/xxx
```

文章最终保存为 Markdown、图片、索引和结构化页面数据，方便搜索、整理和长期积累。用户侧只需要理解两个主场景：**订阅同步**和**微信收藏**。**链接导入**是辅助入口；旧代理历史只作为备用和诊断能力。

## 安装

把项目克隆到 Codex 或 Claude Code 的 Skills 目录：

```bash
# Codex
mkdir -p ~/.codex/skills
git clone https://github.com/Moore-developers/moore-wechat-article-downloader.git \
  ~/.codex/skills/moore-wechat-article-downloader

# Claude Code
mkdir -p ~/.claude/skills
git clone https://github.com/Moore-developers/moore-wechat-article-downloader.git \
  ~/.claude/skills/moore-wechat-article-downloader
```

重新打开 Codex / Claude Code 对话后，直接用自然语言提出下载或同步需求。基础下载只需要 Python 3，不需要 Dashboard、浏览器插件或微信登录。

按需依赖：

- Exporter 历史列表需要扫码登录自己的微信公众号后台；不需要拥有目标公众号。
- 微信收藏和旧代理历史需要 macOS 微信桌面客户端、mitmproxy 及受信任的本地证书。
- 自动切换和恢复系统 HTTP/HTTPS 代理目前只支持 macOS。

## 场景骨架

| 用户场景 | 解决什么问题 | 默认入口 | 说明 |
|---|---|---|---|
| **订阅同步** | 长期跟踪一个或多个公众号，批量同步新文章，按标题、日期或范围下载正文 | Exporter | 默认不抓评论和互动数据 |
| **微信收藏** | 在微信阅读时收藏当前文章、已加载评论和页面互动数据；也为同公众号已同步文章补充短时互动数据 | 微信收藏 | 需要微信桌面端和本地证书，只承诺精选评论 |
| **链接导入** | 已经有文章 URL，想快速归档正文和图片 | 直接下载 | 无需登录，不能发现公众号历史 |
| **旧代理历史** | Exporter 不可用或需要诊断时，从微信旧版历史页被动捕获文章列表 | 备用能力 | 不作为主流程展示 |

评论、阅读数、点赞数、在看数和精选评论不是订阅同步的默认产物。只有用户明确说“评论”“互动”“阅读数”“点赞”“精选评论”等需求时，才进入微信收藏短时凭证流程。

## 快速开始

### 订阅同步：批量跟踪公众号

```text
同步所有已订阅公众号，列出最近 50 篇文章，按公众号和日期整理
订阅并同步「某公众号」
下载「某公众号」最新 20 篇
```

订阅同步是默认主流程，适合“长期关注一批公众号”和“批量整理某个公众号”的场景。

1. 未登录时生成二维码。
2. 扫码登录自己的微信公众号后台。
3. Skill 搜索或读取已订阅公众号，并同步文章列表。
4. 在对话中列出标题、日期和公众号名。
5. 根据你的需求下载最新 N 篇、指定日期范围或标题关键词命中的文章。

登录只用于搜索公众号和读取历史文章列表。Exporter 不获取评论、阅读数、点赞数或收藏数。

如果你已经明确说“下载最新 20 篇”，Skill 会把标题列表作为范围回显，然后直接下载，不会再要求你做一次无意义选择。只有范围不明确时，才让你挑选。

### 订阅同步 + 评论互动：明确要求时才触发

```text
下载「<某公众号>」最新 5 篇文章的评论和互动数据
重新下载「某公众号」最新 20 篇，并补充阅读数、点赞数和精选评论
```

这个流程会拆成两条线并行：

1. Exporter 先同步目标公众号，确定目标文章，并下载正文。
2. 如果当前没有该公众号的微信短时凭证，Skill 会自动开启微信收藏，复制一篇代表文章链接，并提示你粘贴到微信文件传输助手后打开。
3. 你在微信里打开代表文章后，Skill 在短时有效窗口内为同公众号的目标文章批量请求互动指标和**精选评论**。
4. 结果直接写回对应文章的 Markdown `## 页面数据` 区块；重复运行会覆盖旧区块，不会追加重复内容。

边界很重要：这里不承诺全量评论，不承诺完整回复树；只能保存接口或页面实际返回的精选评论和可观察指标。

### 微信收藏：边浏览边保存

```text
开启微信收藏
```

这个场景适合日常阅读：你在微信里正常看文章，遇到值得保留的正文、评论或互动数据时，点击页面里的 `收藏到本地`。

Skill 会开启微信收藏，并重置微信的文章 WebView 进程以加载最新注入代码；微信主进程和登录状态不会退出。在微信文章页等待正文和评论加载完成，然后点击注入按钮。你可以继续浏览并收藏多篇，不需要每篇都回到对话操作。成功注入的文章页会禁用本地缓存，后续重载不会继续复用旧按钮代码。

微信收藏有两个互补能力：

- **当前页收藏**：保存你正在看的这篇文章、页面已加载评论、当前可观察的阅读/点赞/在看等数据。
- **批量互动补充**：打开目标公众号任意一篇文章后，在短时凭证有效窗口内，为该公众号已同步的目标文章补充互动指标和精选评论。

当前常驻服务还没有作为默认启动方式启用；使用时仍需要显式开启和关闭微信收藏会话。

浏览结束后说：

```text
已保存，帮我整理
```

Skill 会一次处理所有尚未整理的收藏：已归档文章会补充页面数据，新文章会生成 Markdown。可保存内容包括：

- 页面已经加载的评论及其昵称、地区或时间、点赞和回复。
- 页面当前暴露的阅读数、点赞数、评论数和在看数。
- 正文、图片 URL 和可观察的页面排版信息。

这些数据受微信页面结构和加载状态影响：评论不保证完整，未暴露的指标会标记为 `missing`，不会猜测。

结束微信收藏时说：

```text
关闭微信收藏
```

Skill 会恢复原来的系统代理，停止本次随机端口的代理进程，并保留已收藏的数据。详细的代理安装、路由检查和安全停止方式见 [`references/troubleshooting.md`](references/troubleshooting.md)。

### 链接导入：已有 URL 快速归档

```text
下载这些文章：
https://mp.weixin.qq.com/s/aaa
https://mp.weixin.qq.com/s/bbb
```

也可以提供包含文章 URL 的 `.txt`、`.csv` 或 `.json` 文件。链接导入无需登录，适合把零散文章快速保存到本地。它只负责正文和图片，不负责公众号发现、历史同步、评论或互动数据。

重复运行时，文章仍保存到对应公众号目录；如果本地已经存在文章，显式刷新或重新下载会覆盖旧文件，SQLite 和 `index.csv` 会记录状态。

### 备用：旧代理历史

只有在明确不想使用 Exporter、Exporter 不可用，或需要诊断微信接口变化时，才使用旧代理历史：

```text
我不登录，用微信获取这个公众号的历史文章：https://mp.weixin.qq.com/s/xxx
```

Skill 会生成公众号旧版历史入口。用微信桌面客户端内置浏览器打开并向下滚动后，Skill 只会列出实际加载到的文章，因此不能保证覆盖全部历史。

此流程会修改系统代理。启用前必须明确确认，结束时由 Skill 恢复原来的代理设置。它不是主流程，只是兼容和诊断兜底。

## 输出

默认按公众号保存：

```text
~/Downloads/wechat-articles/<公众号名>/
├── articles/<文章标题>.md
├── images/<文章标题>/
├── snapshots/              # 使用微信收藏时按需关联
└── index.csv
```

`index.csv` 记录标题、公众号、原文 URL、本地 Markdown、图片数量和下载状态。订阅、文章状态、评论互动状态和任务记录保存在本地 SQLite；运行状态、登录资料和临时快照保存在 `~/.moore/wechat-article-downloader/`，不会混入交付目录。

## 平台与边界

- 订阅同步和微信收藏是主能力；链接导入是辅助入口；旧代理历史是备用和诊断能力。
- 订阅同步默认不抓评论和互动数据；需要用户明确提出评论、互动、阅读数、点赞数或精选评论。
- 只处理用户有权访问的公开文章，不绕过登录、付费墙或私密内容。
- 不在对话中打印 cookie、auth-key、token、pass_ticket 等敏感凭据。
- Exporter 凭据优先保存在 macOS Keychain，内部状态保存在本地 SQLite。
- 页面快照和调试日志会脱敏；不会递归复制原始快照到文章库。
- 项目不提供内容改写、云端托管或 SaaS 服务。

## 开发与文档

三个 CLI 分工如下：

- `scripts/wechat_wizard.py`：自然语言任务、登录和选择 gate 的统一入口。
- `scripts/wechat_exporter.py`：公众号搜索、历史同步、订阅和增量下载。
- `scripts/wechat_downloader.py`：链接导入、微信收藏、旧代理历史和页面快照归档。

运行离线契约测试：

```bash
python3 -m unittest discover -s evals -p 'test_*contract.py'
```
