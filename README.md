# 公众号研究助手（本地版）

**把刷过的公众号，变成可以持续积累的创作素材和竞争情报库。**

内容创作者每天刷几十篇公众号文章，却常常看完就忘、难以复用；
产品经理想长期跟踪竞品选题，却苦于数据分散、评论缺失；
AI 用户需要高质量语料，却缺少干净、结构化、可长期管理的本地输入。

现在，你可以把这些内容沉淀下来：

- 自动同步关注的公众号文章
- 保存正文、图片，以及页面实际提供的精选评论和互动数据
- 转为结构化的本地资料库
- 随时搜索、拆解，或交给 AI 做进一步分析

**目标**：帮助内容创作者、产品研究者和 AI 用户，把零散的公众号阅读转化为系统、可积累、可复用的个人知识资产。

---

## 核心亮点

- **告别信息碎片化**：把公众号文章归档成永久可搜索的本地资料库，不再依赖聊天记录和收藏夹翻找。
- **补齐读者反馈**：除正文外，还可在微信短时会话有效时保存页面实际返回的精选评论、阅读、点赞、在看等互动信号。
- **边看边收藏**：在微信桌面客户端打开文章时，可以通过“收藏到本地”把当前正文、已加载评论和页面数据一起归档。
- **适合深度研究**：Markdown、CSV 和 SQLite 输出可直接用于选题拆解、标题分析、内容复盘和 AI 辅助研究。
- **本地优先**：文章库、索引和凭据保存在本机；除访问微信文章和 Exporter 接口外，不上传到项目自建的云端服务。

**一句话：把别人随手刷过的公众号，变成你可以反复使用的内容情报库。**

## 快速开始

安装到 Codex：

```bash
mkdir -p ~/.codex/skills
git clone https://github.com/Moore-developers/moore-wechat-article-downloader.git \
  ~/.codex/skills/moore-wechat-article-downloader
```

安装到 Claude Code：

```bash
mkdir -p ~/.claude/skills
git clone https://github.com/Moore-developers/moore-wechat-article-downloader.git \
  ~/.claude/skills/moore-wechat-article-downloader
```

重新打开 Codex 或 Claude Code，然后直接描述目标：

```text
同步我关注的公众号，按日期列出最近 50 篇新文章
把「某公众号」最近 20 篇文章收进本地资料库
结合最近 5 篇文章的精选评论和互动数据，判断读者真正买账的点
开启微信收藏会话，边看边把正文、评论和互动信号保存到本地
把这些公众号文章链接导入本地：https://mp.weixin.qq.com/s/xxx
```

## 适用人群

- **内容创作者**：拆解选题、标题、结构和读者反馈，建立自己的素材库。
- **产品经理和研究者**：长期跟踪公众号和竞品内容，比较主题与表达变化。
- **知识管理用户**：把微信内容沉淀到本地资料库、Wiki 或第二大脑。
- **AI 用户**：为 Codex、Claude Code 等工具准备结构化、可追溯的本地输入。

## 核心使用场景

### 1. 同步更新

适合把关注的公众号变成可持续追踪的文章列表。

```text
同步我关注的公众号，按日期列出最近 50 篇新文章
从已订阅公众号里筛出最近值得精读的 10 篇
整理最近一周的新文章，按公众号分组
```

默认只下载正文和图片，不抓取评论与互动数据。

### 2. 公众号研究

适合分析一个公众号的选题方式、内容结构和读者反馈。

```text
看「某公众号」最近 10 篇文章，判断它值不值得长期跟踪
拆「某公众号」最近 10 篇文章的选题、标题和内容结构
基于最近 20 篇文章和精选评论，提炼可以复用的选题方法
```

研究素材包括：

- **正文**：作者如何立题、组织结构和表达观点。
- **精选评论**：读者认同、反对或继续追问的内容。
- **互动信号**：页面实际提供的阅读、点赞、在看、评论等数据。

### 3. 微信收藏会话

适合在微信里阅读时，把当前文章现场保存到本地。

```text
开启微信收藏会话，我要边看文章边保存正文、评论和互动信号
```

打开文章后点击页面里的“收藏到本地”，可以保存：

- 当前文章正文和图片
- 页面已经加载的评论
- 页面当前暴露的互动数据
- 页面结构和样式摘要

评论和互动数据依赖当前微信页面及短时有效凭据，不保证每篇文章都能获得相同字段。

### 4. 链接归档

适合把聊天记录、收藏夹或文件中的零散文章链接统一归档。

```text
把这些公众号文章链接导入本地：https://mp.weixin.qq.com/s/xxx
把群里发来的公众号文章链接按公众号归档
把一批文章链接收进本地，后面用于选题分析
```

## 输出文件结构

默认按公众号保存到：

```text
~/Downloads/wechat-articles/<公众号名>/
```

典型结构：

```text
<公众号名>/
├── index.csv
├── articles/
│   └── <文章标题>.md
└── images/
    └── <文章标题>/
        └── 001.<扩展名>
```

本地运行状态保存在：

```text
~/.moore/wechat-article-downloader/
```

其中包括 SQLite 索引、任务状态、页面快照和脱敏后的运行记录。

## 与普通工具的区别

| 维度 | 普通导出工具 | 公众号研究助手 |
|---|---|---|
| 产品入口 | 需要理解脚本和参数 | 直接用自然语言描述同步、研究、收藏或归档目标 |
| 内容范围 | 以正文批量导出为主 | 正文、图片，以及页面实际提供的精选评论和互动信号 |
| 当前页收藏 | 通常不支持 | 可在微信里边看边保存当前文章现场 |
| 输出方式 | 按任务分散输出 | 按公众号维护长期 Markdown、图片和 CSV 索引 |
| 本地状态 | 偏一次性任务 | 使用 SQLite 管理公众号、文章、下载和互动状态 |
| 隐私边界 | 取决于具体服务 | 本地优先，不提供 SaaS 或项目自建云端托管 |

它不是“导出后就结束”的下载脚本，而是一套围绕同步、收藏、研究和归档建立的本地工作流。

## 安装与使用

### 运行要求

- Python 3
- 需要访问公开的微信文章页面
- Exporter 历史列表需要扫码登录自己的微信公众号后台
- 微信收藏和代理历史需要 macOS 微信桌面客户端、本地证书和 mitmproxy
- 自动切换和恢复系统代理目前只支持 macOS

### 更新

```bash
cd ~/.codex/skills/moore-wechat-article-downloader
git pull
```

如果安装在 Claude Code 的 Skills 目录，把路径替换为：

```text
~/.claude/skills/moore-wechat-article-downloader
```

## 边界与隐私

- 只处理用户当前有权访问的公开文章，不绕过登录、付费墙或私密内容。
- 只承诺页面或接口实际返回的数据，不承诺全量评论或完整回复树。
- 精选评论和互动数据依赖短时微信会话，字段可能缺失或失效。
- 不打印或对外输出 cookie、token、auth-key、pass_ticket 等敏感值。
- 文章库、索引和凭据保存在本机，但获取内容时仍会访问微信和配置的 Exporter 服务。
- 不提供 SaaS、云端托管或内容改写服务。

## 开发与贡献

主要入口：

- `scripts/wechat_wizard.py`：自然语言任务、登录和选择入口
- `scripts/wechat_exporter.py`：公众号搜索、历史同步、订阅和增量下载
- `scripts/wechat_downloader.py`：链接导入、微信收藏、代理历史和页面快照归档

运行离线契约测试：

```bash
python3 -m unittest discover -s evals -p 'test_*contract.py'
```
