# SKILL 应用场景重新设计

## 设计原则

**核心变更**：Exporter 模式是获取历史列表的**默认**和**首选**方案，代理模式降级为备选。

## 重新定义的三种场景

### 场景 1：直接下载已知文章
**用户有什么**：一个或多个公众号文章 URL  
**用户想要**：下载这些文章的 Markdown + 图片

**触发条件**：
- 用户提供了完整的 `mp.weixin.qq.com/s/...` URL
- 或提供了包含 URL 列表的文件

**核心命令**：
```bash
wechat_wizard.py run "下载这篇文章：<url>"
wechat_wizard.py run "下载这些文章：<urls-or-file>"
```

**无需**：登录、代理、历史列表

---

### 场景 2：获取并下载公众号历史文章（默认：Exporter 模式）
**用户有什么**：公众号名称 / 一篇样例文章 URL / 公众号关键词  
**用户想要**：
1. 找到这个公众号
2. 列出历史文章
3. 选择并下载部分或全部

#### 2A. Exporter 模式（**默认首选**）
**适用条件**：
- 用户愿意扫码登录（一次登录，4天有效）
- 用户想要可靠的历史列表
- 用户需要搜索公众号、管理多个公众号

**优势**：
- ✅ 稳定的 API，不依赖代理抓包
- ✅ 支持按公众号名搜索
- ✅ 可管理多个公众号
- ✅ SQLite 持久化，可复用

**核心流程**：
```bash
# 一句话启动
wechat_wizard.py run "下载公众号「某公众号」最新 20 篇"

# wizard 会自动处理：
# - 检查登录状态（未登录→引导扫码）
# - 搜索公众号（多个结果→让用户选择）
# - 同步文章列表
# - 下载选中文章
```

**中断点**：
- `need_login` → 用户扫码 → `login-complete`
- `need_account_choice` → 用户选公众号 → `resume --choice`
- `need_article_choice` → 用户选文章 → `resume --choice`

#### 2B. 代理模式（**备选方案**）
**适用条件**：
- 用户不想登录公众号后台
- 或 Exporter 登录失败
- 用户只需要一次性抓取一个公众号

**劣势**：
- ❌ 需要安装 mitmproxy 和信任证书
- ❌ 需要切换系统代理（macOS）
- ❌ 需要用微信内置浏览器手动滚动
- ❌ 不稳定，容易被微信更新影响

**触发方式**：
```bash
# 用户明确说"不登录"或"用代理"
wechat_wizard.py run "用代理模式抓取这个公众号的历史：<url>"

# 或 Exporter 失败后降级
```

**说明**：
- 代理模式不应该作为用户的第一选择
- SKILL 应该主动推荐 Exporter 模式
- 只有在明确场景下才使用代理

---

### 场景 3：批量管理和定期同步
**用户有什么**：需要持续跟踪多个公众号  
**用户想要**：
- 管理公众号列表
- 定期同步最新文章
- 查看合集
- 自定义字段

**解决方案**：Exporter 模式 + 本地管理页（可选）

**核心流程**：
```bash
# 添加多个公众号
wechat_wizard.py run "搜索并添加公众号「某公众号」「小报童」"

# 同步所有已添加的公众号
wechat_wizard.py run "同步所有公众号的最新文章"

# 图形化管理（可选）
wechat_exporter.py exporter-server-start --port 8765
```

## 场景判断决策树

```
用户输入
  │
  ├─ 包含完整 URL？
  │   └─ 是 → **场景 1：直接下载**
  │
  ├─ 提到"公众号名"或"搜索公众号"？
  │   └─ 是 → **场景 2A：Exporter 模式**
  │
  ├─ 提供了一篇样例文章 URL + 想要历史列表？
  │   ├─ 默认 → **场景 2A：Exporter 模式**
  │   │         （从 URL 提取 biz → 搜索公众号）
  │   └─ 明确说"不登录"或"用代理" → **场景 2B：代理模式**
  │
  ├─ 提到"管理多个公众号"或"定期同步"？
  │   └─ 是 → **场景 3：批量管理**
  │
  └─ 不确定？
      └─ 询问用户意图
```

**优先级 P1（50-150 行）**：
1. 每种场景的详细流程

3. 常见选项（并发、重试、输出目录）

**优先级 P2（150+ 行或独立文档）**：
1. 完整命令参考
2. 参数详解

## 对用户交互的影响

### 当前问题
用户说："我想下载公众号「某公众号」的历史文章"

AI 可能会：
1. 引导用户安装 mitmproxy
2. 设置代理
3. 用微信浏览器打开链接...

→ **体验差，步骤多**

### 优化后
用户说："我想下载公众号「某公众号」的历史文章"

AI 回复：
```
好的，我会用 Exporter 模式帮你获取「某公众号」的历史文章。

这需要一次性扫码登录（4天有效），之后就能直接搜索和下载任何公众号。

开始吗？
```

→ **体验好，一句话启动**

## 下载优化建议

基于你提到的"下载方面也需要优化"，建议关注：

### 下载性能优化
- 当前：默认串行，保守
- 建议：智能并发（根据网络质量自适应）

### 下载结果验证
- 当前：需要手动运行 `validate_outputs.py`
- 建议：下载完成后自动验证并报告

### 断点续传
- 当前：支持（同 task-id 跳过已成功 URL）
- 建议：在 SKILL 中明确说明这个能力

### 输出格式
- 当前：只有 Markdown + 图片
- 建议：支持 PDF、HTML 等（如果需要）
